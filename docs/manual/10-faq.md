## アカウント管理

### Q1: 初めてログインするにはどうすればいいですか？

**A:** システムには以下のデフォルトアカウントが用意されています：

| ユーザー名 | パスワード | ロール | 説明 |
|----------|----------|--------|-----|
| admin | Admin123! | 管理者 | すべての機能を利用可能 |
| testuser | Password123! | 一般ユーザー | 閲覧のみ可能 |

ログインページでユーザー名とパスワードを入力してログインしてください。

### Q2: パスワードを変更できますか？

**A:** はい、可能です。ただし、現在のバージョンでは管理者による直接変更のみサポートしています。将来のバージョンでは、ユーザー自身でパスワード変更ができる機能を追加予定です。

### Q3: ログアウトするにはどうすればいいですか？

**A:** 画面右上のユーザー名をクリックすると、ログアウトオプションが表示されます。ログアウト後は自動的にログインページに戻ります。

### Q4: ログインしたままの状態は保持されますか？

**A:** はい、セッション情報が保存されるため、ブラウザを閉じても次回アクセス時にログイン状態が維持されます。ただし、セキュリティのため、一定期間操作がない場合は自動的にログアウトされます。

## 権限とロール

### Q5: 従業員情報を編集できるのは誰ですか？

**A:** **管理者（Admin）ロール**を持つユーザーのみが従業員情報の作成、編集、削除を行えます。一般ユーザー（User）ロールは従業員情報の閲覧のみ可能です。

### Q6: 自分のロールを確認するにはどうすればいいですか？

**A:** ログイン後、画面右上に表示されるユーザー名の横にロールが表示されます。また、管理者の場合は「従業員を追加」ボタンなどの編集機能が表示されます。

### Q7: 一般ユーザーから管理者に昇格できますか？

**A:** ロールの変更は、現在のところ管理者による直接的なデータベース操作またはシステム管理者への依頼が必要です。将来のバージョンでは、UI上でロール管理機能を追加予定です。

### Q8: なぜ「従業員を追加」ボタンが表示されないのですか？

**A:** このボタンは管理者（Admin）ロールのユーザーにのみ表示されます。一般ユーザーでログインしている場合は表示されません。管理者権限が必要な場合は、システム管理者に連絡してください。

## 従業員管理

### Q9: 新しい従業員を登録するにはどうすればいいですか？

**A:** 管理者としてログインし、以下の手順で登録できます：

1. 従業員一覧ページにアクセス
2. 「従業員を追加」ボタンをクリック
3. 必要な情報を入力（名前、メールアドレス、部署など）
4. 「保存」ボタンをクリック

すべての必須フィールドを入力する必要があります。

### Q10: 従業員情報を編集するにはどうすればいいですか？

**A:** 管理者としてログインし、以下の手順で編集できます：

1. 従業員一覧ページで編集したい従業員を探す
2. 該当する従業員の「編集」ボタンをクリック
3. 情報を修正
4. 「保存」ボタンをクリック

編集が完了すると、変更内容が即座に反映されます。

### Q11: 従業員を削除することはできますか？

**A:** はい、管理者権限があれば削除可能です。ただし、削除操作は取り消せないため、慎重に実行してください。削除すると、その従業員に関連するすべてのデータ（勤怠記録など）も影響を受ける可能性があります。

### Q12: 従業員情報の検索はできますか？

**A:** はい、従業員一覧ページの検索ボックスを使用して、名前やメールアドレスで従業員を検索できます。部分一致検索がサポートされているため、名前の一部を入力するだけで候補が絞り込まれます。

### Q13: 従業員の詳細情報はどこで確認できますか？

**A:** 従業員一覧ページで従業員名をクリックすると、詳細ページに遷移します。詳細ページでは、基本情報に加えて勤怠履歴や通知履歴なども確認できます。

## 勤怠管理

### Q14: 出勤記録はどのように登録しますか？

**A:** 勤怠管理ページにアクセスし、以下の手順で登録できます：

1. 「出勤記録」ボタンをクリック
2. 従業員を選択
3. 出勤時刻を入力（デフォルトは現在時刻）
4. 「記録」ボタンをクリック

出勤記録は即座にシステムに保存されます。

### Q15: 退勤時刻を記録し忘れた場合はどうすればいいですか？

**A:** 管理者権限があれば、後から勤怠記録を編集して退勤時刻を追加できます。勤怠一覧ページから該当する記録を選択し、退勤時刻を入力してください。

### Q16: 勤務時間はどのように計算されますか？

**A:** 勤務時間は出勤時刻と退勤時刻の差分から自動的に計算されます。例えば、9:00に出勤して18:00に退勤した場合、勤務時間は9時間として記録されます。

### Q17: 月次の勤怠集計はどこで確認できますか？

**A:** 各従業員の詳細ページまたは勤怠管理ページで、月次集計を確認できます。以下の情報が表示されます：

- 総出勤日数
- 総勤務時間
- 平均勤務時間
- 遅刻回数（9:00以降の出勤回数）

### Q18: 休暇申請はどのように行いますか？

**A:** 勤怠管理ページの「休暇申請」セクションから申請できます：

1. 「休暇申請」ボタンをクリック
2. 休暇の種類を選択（有給休暇、病気休暇など）
3. 開始日と終了日を入力
4. 理由を記入
5. 「申請」ボタンをクリック

申請後は承認待ちステータスとなり、管理者が承認または却下を行います。

### Q19: 休暇申請の承認状況はどこで確認できますか？

**A:** 勤怠管理ページの休暇申請一覧で、各申請のステータス（承認待ち、承認済み、却下、キャンセル済み）を確認できます。通知機能が有効な場合、承認または却下時に通知も届きます。

### Q20: 遅刻はどのように判定されますか？

**A:** 標準の勤務開始時刻である9:00以降に出勤記録が登録された場合、自動的に遅刻としてカウントされます。月次集計で遅刻回数を確認できます。

## 通知機能

### Q21: 通知はどこで確認できますか？

**A:** 画面上部のベルアイコンをクリックすると、通知一覧が表示されます。また、通知ページにアクセスすることで、すべての通知の詳細を確認できます。

### Q22: どのような場合に通知が届きますか？

**A:** 以下のイベント発生時に通知が届きます：

- 従業員情報が作成・更新された時
- 勤怠記録が登録された時
- 休暇申請が承認・却下された時
- 勤怠異常が検知された時
- システムからの重要なお知らせ

### Q23: 通知を削除することはできますか？

**A:** はい、各通知の横にある削除ボタンをクリックすることで、通知を削除できます。また、「すべて削除」ボタンで一括削除も可能です。

### Q24: 通知がリアルタイムで届かない場合はどうすればいいですか？

**A:** 以下を確認してください：

1. ブラウザを最新の状態に保つ
2. ページを再読み込みする
3. ネットワーク接続を確認する

問題が解決しない場合は、システム管理者に連絡してください。

## トラブルシューティング

### Q25: 「401 Unauthorized」エラーが表示されます

**A:** このエラーは認証情報が正しくない場合に表示されます。以下を試してください：

1. ログアウトして再度ログインする
2. ブラウザのキャッシュとCookieをクリアする
3. 正しいユーザー名とパスワードを使用しているか確認する

問題が続く場合は、システム管理者に連絡してください。

### Q26: 「403 Forbidden」エラーが表示されます

**A:** このエラーは、実行しようとした操作に対する権限がない場合に表示されます。管理者権限が必要な操作を実行しようとしていないか確認してください。必要に応じて、システム管理者に権限の付与を依頼してください。

### Q27: ページの読み込みが遅い場合はどうすればいいですか？

**A:** 以下を試してください：

1. ブラウザのキャッシュをクリアする
2. 不要なブラウザタブを閉じる
3. ネットワーク接続を確認する
4. ブラウザを再起動する

大量のデータを処理している場合、読み込みに時間がかかることがあります。

### Q28: データが保存されない、または更新されない場合

**A:** 以下を確認してください：

1. すべての必須フィールドに入力されているか
2. 入力形式が正しいか（メールアドレス、日付など）
3. ネットワーク接続が安定しているか
4. 管理者権限があるか（編集操作の場合）

エラーメッセージが表示されている場合は、その内容を確認してください。

### Q29: 画面が正しく表示されない場合

**A:** 以下を試してください：

1. ブラウザを最新バージョンに更新する
2. ブラウザのキャッシュをクリアする
3. 別のブラウザで試してみる

推奨ブラウザ：Chrome、Edge、Firefox の最新版

### Q30: システムがダウンしているかどうか確認するには？

**A:** Aspireダッシュボード（開発環境）または監視ツール（本番環境）で、すべてのサービスのヘルスチェック状態を確認できます。すべてのサービスが「Healthy」状態であれば正常に稼働しています。

## システム全般

### Q31: このシステムの主な機能は何ですか？

**A:** 従業員管理システムには以下の主な機能があります：

- **従業員管理**: 従業員情報の作成、閲覧、編集、削除
- **勤怠管理**: 出退勤記録、勤怠履歴、月次集計
- **休暇管理**: 休暇申請、承認フロー
- **通知機能**: リアルタイム通知、イベント通知
- **認証・認可**: ロールベースのアクセス制御

### Q32: どのブラウザで利用できますか？

**A:** 以下の最新版ブラウザをサポートしています：

- Google Chrome
- Microsoft Edge
- Mozilla Firefox
- Safari（macOS/iOS）

最良の体験のため、常に最新版のブラウザを使用することをお勧めします。

### Q33: モバイルデバイスから利用できますか？

**A:** はい、システムはレスポンシブデザインを採用しており、スマートフォンやタブレットからもアクセスできます。ただし、一部の機能は大きな画面での利用を前提としているため、デスクトップブラウザでの利用を推奨します。

### Q34: データのバックアップはどのように行われますか？

**A:** システムのバックアップは、本番環境では定期的に自動実行されます。開発環境ではSQLiteデータベースファイルが使用されており、`data/` ディレクトリに保存されています。

### Q35: システムの稼働状況を監視することはできますか？

**A:** はい、以下の方法で監視できます：

- **開発環境**: Aspireダッシュボードでリアルタイムの監視、ログ、トレースを確認
- **本番環境**: Application Insights（または類似の監視ツール）で監視

管理者権限があれば、これらのツールにアクセスできます。

### Q36: 複数のユーザーが同時に利用できますか？

**A:** はい、システムはマイクロサービスアーキテクチャで構築されており、複数のユーザーが同時にアクセスして利用できます。ただし、同じデータを複数のユーザーが同時に編集する場合は注意が必要です。

### Q37: システムのメンテナンス時間はありますか？

**A:** 定期メンテナンスが予定されている場合は、事前に通知が届きます。緊急メンテナンスの場合は、可能な限り早く通知されます。メンテナンス中はシステムにアクセスできない場合があります。

### Q38: 新機能のリクエストやバグ報告はどこで行えますか？

**A:** GitHub リポジトリの Issues セクションで、新機能のリクエストやバグ報告を受け付けています。詳細な情報を提供していただくことで、より早く対応できます。

### Q39: システムの詳細な技術情報はどこで確認できますか？

**A:** 以下のドキュメントを参照してください：

- [Getting Started](../getting-started.md) - セットアップと実行方法
- [アーキテクチャ概要](../architecture.md) - システム設計
- [開発ガイド](../development-guide.md) - 開発者向け情報
- [データベース管理](../database.md) - データベース設計
- [Aspireダッシュボード](../aspire-dashboard.md) - 監視と管理

### Q40: サポートが必要な場合はどうすればいいですか？

**A:** 以下の方法でサポートを受けられます：

1. このFAQで問題が解決しないか確認する
2. 各種ドキュメントを参照する
3. システム管理者に連絡する
4. GitHub Issues で質問を投稿する

## 用語集

| 用語 | 説明 |
|------|------|
| **管理者（Admin）** | すべての機能にアクセスできる権限を持つユーザーロール |
| **一般ユーザー（User）** | 閲覧のみ可能な権限を持つユーザーロール |
| **出勤記録（Check-in）** | 従業員の勤務開始時刻を記録する操作 |
| **退勤記録（Check-out）** | 従業員の勤務終了時刻を記録する操作 |
| **勤務時間（Work Hours）** | 出勤時刻と退勤時刻の差分から計算される時間 |
| **月次集計（Monthly Summary）** | 1ヶ月間の勤怠データを集計した情報 |
| **休暇申請（Leave Request）** | 休暇を取得するための申請 |
| **通知（Notification）** | システムからユーザーへの情報伝達メッセージ |
| **Aspireダッシュボード** | .NET Aspireが提供する監視・管理ツール |
| **マイクロサービス** | 独立して動作する小さなサービスで構成されたアーキテクチャ |
